# @marianmeres/tree - LLM Context File

## Package Overview

A TypeScript/JavaScript generic tree data structure library providing common traversal, lookup, and node manipulation operations. Designed as a base class for hierarchical data management.

## Core Classes

### TreeNode<T>

Represents a single node in the tree. Generic type T is the value type stored in nodes.

#### Properties (getters)
- `id: string` - Auto-generated unique identifier (prefixed with "n", safe for HTML IDs)
- `value: T` - The stored value
- `parent: TreeNode<T> | null` - Parent node reference
- `children: TreeNode<T>[]` - Array of direct child nodes
- `depth: number` - Level in tree (root = 0)
- `isLeaf: boolean` - True if no children
- `isRoot: boolean` - True if no parent
- `siblings: TreeNode<T>[]` - Nodes sharing same parent (includes self)
- `siblingIndex: number` - Position within siblings array
- `path: TreeNode<T>[]` - Ancestors from root to parent (self excluded)
- `root: TreeNode<T> | null` - Reference to root node
- `tree: Tree<T> | null` - Reference to owning Tree instance
- `readonly: boolean` - Whether modifications are blocked

#### Methods
- `appendChild(valueOrNode: T | TreeNode<T>): TreeNode<T>` - Add child node
- `removeChild(id: string): TreeNode<T>` - Remove child by ID
- `replaceChild(id: string, valueOrNode: T | TreeNode<T>): TreeNode<T> | false` - Replace child
- `resetChildren(valuesOrNodes?: (T | TreeNode<T>)[]): TreeNode<T>` - Clear and set new children
- `previousSibling(): TreeNode<T> | null` - Get preceding sibling
- `nextSibling(): TreeNode<T> | null` - Get following sibling
- `moveSiblingIndex(toIndex: number): TreeNode<T>` - Reorder within siblings
- `contains(id: string, maxDepth?: number): boolean` - Check if descendant exists
- `has(value: T, maxDepth?: number, compareFn?): boolean` - Check if value exists in descendants
- `matches(valueOrPropValue, propName?, compareFn?): TreeNode<T> | null` - Check if node matches criteria
- `findAllBy(valueOrPropValue, propName?, maxDepth?, compareFn?): TreeNode<T>[]` - Find matching descendants
- `deepClone(): TreeNode<T>` - Create deep copy with new IDs
- `toJSON(): TreeNodeDTO<T>` - Serialize to DTO
- `toString(): string` - Debug string with indentation

#### Internal Methods (double-underscore prefix, not for public use)
- `__setId(id: string)` - Set node ID
- `__setParent(parent: TreeNode<T> | null)` - Set parent reference
- `__setTree(tree: Tree<T> | null)` - Set tree reference
- `__setReadonly(flag: boolean)` - Set readonly flag
- `__syncChildren()` - Synchronize child references recursively

### Tree<T>

Container class managing the tree structure with a single root node.

#### Constructor
`new Tree<T>(root?: TreeNode<T> | null, readonly?: boolean)`

#### Properties
- `root: TreeNode<T> | null` - The root node
- `readonly: boolean` - Whether tree is immutable

#### Static Methods
- `Tree.factory<T>(dump: string | TreeNodeDTO<T>, readonly?: boolean): Tree<T>` - Create from serialized data

#### Traversal Methods (Generators)
- `preOrderTraversal(node?): Generator<TreeNode<T>>` - Depth-first, pre-order (NLR)
- `postOrderTraversal(node?): Generator<TreeNode<T>>` - Depth-first, post-order (LRN)
- `levelOrderTraversal(node?): Generator<TreeNode<T>>` - Breadth-first (BFS)

#### Lookup Methods
- `find(id: string): TreeNode<T> | null` - Find node by ID
- `findAllBy(value, propName?, maxDepth?, compareFn?): TreeNode<T>[]` - Find by value/property
- `findLCA(node1Id: string, node2Id: string): TreeNode<T> | null` - Lowest common ancestor
- `contains(id: string, maxDepth?: number): boolean` - Check existence by ID
- `has(value: T, maxDepth?, compareFn?): boolean` - Check existence by value

#### Modification Methods
- `appendChild(valueOrNode: T | TreeNode<T>): TreeNode<T>` - Add to root (or set as root if empty)
- `insert(parentNodeId: string, value: T): TreeNode<T>` - Insert under specific parent
- `remove(id: string): Tree<T>` - Remove node and subtree
- `move(srcNodeId: string, targetNodeId: string): TreeNode<T>` - Move subtree to new parent
- `copy(srcNodeId: string, targetNodeId: string): TreeNode<T>` - Copy subtree to new parent

#### Serialization Methods
- `dump(): string` - Serialize to JSON string
- `restore(dump: string | TreeNodeDTO<T>): Tree<T>` - Restore from serialized data
- `toJSON(): TreeNodeDTO<T> | undefined` - Get DTO representation

#### Utility Methods
- `size(from?: TreeNode<T>): number` - Count nodes (optionally from subtree)
- `toString(): string` - Debug string showing tree structure

### TreeNodeDTO<T>

Interface for serialization/deserialization:
```typescript
interface TreeNodeDTO<T> {
  id: string;
  value: T;
  children: TreeNodeDTO<T>[];
}
```

## Key Features

1. **Generic typing** - T parameter allows any value type (primitives, objects, etc.)
2. **Unique IDs** - Auto-generated IDs safe for HTML element use
3. **Full serialization** - Complete dump/restore via JSON
4. **Three traversal modes** - Pre-order, post-order, level-order (all as generators)
5. **LCA algorithm** - Lowest common ancestor lookup
6. **Readonly mode** - Immutable tree support
7. **Sibling operations** - Navigation and reordering
8. **Deep cloning** - Full subtree copy with new IDs
9. **Flexible search** - By ID, value, or object property with optional depth limits

## Traversal Order Reference

For tree:
```
      F
    /   \
   B     G
  / \     \
 A   D     I
    / \     \
   C   E     H
```

- Pre-order: F B A D C E G I H
- Post-order: A C E D B H I G F
- Level-order: F B G A D I C E H

## Usage Patterns

### Creating a tree
```typescript
const tree = new Tree<string>();
const root = tree.appendChild('Root');
const child = root.appendChild('Child');
// or
const tree2 = new Tree(new TreeNode('Root'));
```

### Iterating nodes
```typescript
for (const node of tree.preOrderTraversal()) {
  console.log(node.value);
}
```

### Serialization round-trip
```typescript
const dump = tree.dump();
const restored = Tree.factory<string>(dump);
// or
const restored2 = new Tree<string>().restore(dump);
```

### Finding nodes
```typescript
const node = tree.find(someId);
const nodes = tree.findAllBy('value');
const nodes2 = tree.findAllBy('propValue', 'propertyName'); // for object values
```

## Constraints and Invariants

1. Tree has exactly 0 or 1 root node
2. Each node belongs to at most one tree
3. Node IDs are unique within a tree
4. Moving a node to its own descendant is prohibited
5. Readonly trees throw errors on any modification attempt
6. Self-containment check returns false (node does not contain itself)

## Error Conditions

Throws Error when:
- Missing required ID argument
- Node not found during operations
- Attempting to modify readonly tree/node
- Recursive reference during move (moving to own descendant)
- Moving node to itself
- Appending node from different tree
- Appending existing sibling
